## Что такое система контроля версий?

**Система контроля версий** (Version Control System, VCS) — это инструмент, который записывает изменения в файле или наборе файлов, чтобы позже можно было вернуться к определённым версиям. Git — самая популярная распределённая система контроля версий.

## Зачем нужен Git?

- Отслеживание истории — видно, кто, когда и что изменил
- Ветвление — можно работать над новыми функциями отдельно
- Командная работа — несколько человек могут работать над одним проектом
- Резервное копирование — код хранится не только локально
- Откат изменений — легко вернуться к работающей версии

## Основные команды Git

### Настройка

git config --global user.name "Ильдар Садыков"
git config --global user.email "sad.ildar0406@gmail.com"

### Создание репозитория

git init
git clone https://github.com/username/repository.git

### Ежедневная работа

git status
git add имя_файла.txt
git add .
git commit -m "Сообщение с описанием изменений"
git push origin main

### Работа с историей

git log
git log --oneline
git diff

### Пример рабочего процесса


git status
git add .
git commit -m "Добавил новую функцию"
git push
